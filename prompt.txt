RUBRICA DO TIPO A — Classificação com if/elif/else
══════════════════════════════════════════════════════════════

Critério 1 — Leitura e tipagem (5 pts)
  - Lê os valores com input()
  - Converte para o tipo correto (int ou float)

Critério 2 — Estrutura condicional if/elif/else (30 pts)
  Para cada variável (proteína, carboidrato, gordura ou equivalentes):
  - Usa if/elif/else corretamente
  - Compara com os limites corretos (< e <=/>= ou equivalente)
  - Não há sobreposição nem lacuna entre as faixas
  - Produz o texto de saída correto para cada categoria
  Dê crédito parcial proporcional ao número de variáveis corretas.

Critério 3 — Classificação final e saída (15 pts)
  - Determina corretamente "Equilibrada" / "Desequilibrada"
    (Equilibrada = TODOS adequados; Desequilibrada = AO MENOS UM fora)
  - Imprime cada nutriente no formato correto
  - Imprime a avaliação final

Erros comuns TIPO A — verifique explicitamente:
  - Condição de equilíbrio errada:
      ERRADO:  if (a, b, c) == 2   ← compara tupla com inteiro, sempre False
      CORRETO: if a == 2 and b == 2 and c == 2
  - Impressão antes de converter o código numérico em texto
  - Uso de = em vez de == na comparação
  - Limites com sobreposição (ex: < 44 e <= 44 na mesma variável)

══════════════════════════════════════════════════════════════
RUBRICA DO TIPO B — Acumulação com laço
══════════════════════════════════════════════════════════════

Critério 1 — Leitura e tipagem (5 pts)
  - Lê os dois valores com input()
  - Converte para int

Critério 2 — Laço com filtro e acumulador (35 pts)
  a) Iteração correta (7 pts)
     - Usa for/while sobre o intervalo correto
     - Inclui AMBOS os extremos: range(a, b+1) ou equivalente
       ERRO COMUM: range(a, b) exclui b

  b) Filtro correto (10 pts)
     - Testa paridade com i % 2 == 0 (par) ou i % 2 != 0 (ímpar)
       ERRO COMUM: i // 2 == 0 testa se i < 2, não paridade

  c) Operação correta (10 pts)
     - Calcula i ** 2 (quadrado do número)
     - Aplica += (soma) ou *= (produto) conforme o código

  d) Inicialização do acumulador (8 pts)
     - Soma: inicializar com 0
     - Produto: inicializar com 1
       ERRO COMUM: produto = 0 → resultado sempre zero

Critério 3 — Saída (10 pts)
  - Imprime o resultado no formato correto

Erros comuns TIPO B — verifique explicitamente:
  - range(a, b) em vez de range(a, b+1) → exclui o último valor
  - i // 2 == 0 em vez de i % 2 == 0 → lógica de paridade errada
  - produto = 0 em vez de produto = 1 → produtório sempre zero
  - acumular i em vez de i**2 → esqueceu elevar ao quadrado
